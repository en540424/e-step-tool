# 給与台帳 ExcelJS 出力：安全手順（E-STEP-TOOL）

## Purpose（目的）
- 給与台帳のExcel出力で、テンプレ破損・数式消失・列幅崩れ等の事故を防ぐ
- “再現できる出力” を保証し、修正時に原因追跡できるようにする

## Steps（手順）
1) 変更対象を分離する
   - UI変更と出力ロジック変更は同一コミットに混ぜない
   - まずは「出力に影響する変更だけ」を切り出す

2) テンプレの読み込みは “未加工テンプレ” を基準にする
   - 出力用WBと、基準テンプレWBを分ける（テンプレ汚染を防ぐ）

3) 1人目の出力でテンプレを汚さない
   - “テンプレWS” を直接編集しない
   - 1人分の出力は必ずコピー先WSで行う

4) シート複製はセル単位コピー方式を優先する
   - 列幅/行高/スタイル/結合/数式 をコピーする
   - clone方式（丸ごとコピー）で崩れる場合は使用禁止

5) 数式・結合・列幅の診断を行う
   - 出力後に「数式が残っているか」「結合が一致しているか」を確認する
   - 重要セル（合計・課税合計・差引支給など）は必ずチェックする

6) 出力結果を最小サンプルで確認する
   - 1人だけ / 2人 / 端数が出るケース の3パターンで出す
   - 差分が出たら decision-log に “仕様変更かバグか” を記録する

## Pitfalls & Guards（事故パターンと防止策）
- 事故：1人目はOKだが2人目以降で列幅・罫線が崩れる
  - 対策：常に “未加工テンプレ” から複製する（汚れたWSを複製しない）

- 事故：数式が値で上書きされて戻らない
  - 対策：数式セルは上書き禁止リストで管理 or 診断スクリプトで検出

- 事故：結合が欠けてレイアウトが崩れる
  - 対策：セルコピー時に mergeCells を必ず同期する

- 事故：テンプレファイル自体が壊れて以後すべて崩れる
  - 対策：テンプレはGit管理し、編集する場合は別ブランチ＋小コミット

## Done criteria（完了条件）
- 1人 / 2人 / 端数ケース の出力がすべてOK
- 合計系（課税合計/控除合計/差引支給）の数式が残っている
- 列幅・結合・罫線が “全員同一” で出力される
- 変更理由が必要な場合は decision-log に記録済み
