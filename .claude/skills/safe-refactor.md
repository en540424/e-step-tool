# Skill: safe-refactor（最小変更・破壊禁止）

## 目的
既存仕様を壊さずに、最小変更で修正する。
勝手に全面置換・リファクタリングしない。

---

## 絶対ルール
- 既存動作を変える可能性がある変更は禁止（必要なら提案→確認→分岐）
- 変更は必ず「差分（diff）」で提示し、適用前に影響範囲を列挙
- 触るファイルは最大3つまで（超える場合は分割）
- 推測実装は禁止（不明点は確認リスト化）
- 既存命名規則・フォルダ構成を尊重

---

## 実行前チェック（必須）
- `git status`
- `git branch --show-current`
- `npm run lint`（ある場合）
- `npm test`（ある場合）

---

## 手順
1) 目的を1行で明確化
2) 影響範囲（触るファイル/関数/画面）を列挙
3) リスク（壊れやすい箇所）を列挙
4) 実装案を小さく分割
5) diffを提示（コピペ可能形式）
6) 実行コマンドを提示

---

## 出力形式
- 影響範囲
- リスク
- 手順
- diff
- 確認コマンド
